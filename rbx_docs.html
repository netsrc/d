Luau Language

Language

Luau is Roblox's fast, safe, gradually-typed scripting language — a strict superset of Lua 5.1. All Roblox scripts are written in Luau. It adds optional typing, performance improvements, and syntax extensions not in standard Lua.

Type System

Gradual type annotations with full inference and generics.

Performance

Compiled to bytecode with JIT-optimised native codegen for hot paths.

Safety

Sandboxed per-script environment — no OS or filesystem access by default.

Compatibility

Superset of Lua 5.1 — most Lua libs work with minor tweaks.

Variables & Types

variables.luau
local name: string = "Player1"
local score: number = 0
local alive: boolean = true
local data: {string: number} = {}

-- String interpolation (Luau exclusive)
print(`Hello, {name}! Score: {score}`)

-- Compound assignment operators
score += 10
score *= 2
score -= 3

Functions & Generics

functions.luau
local function add(a: number, b: number): number
    return a + b
end

-- Generic function
local function first<T>(arr: {T}): T
    return arr[1]
end

-- Type alias
type PlayerData = {
    name: string,
    level: number,
    coins: number
}

OOP via Metatables

oop.luau
local Enemy = {}
Enemy.__index = Enemy

function Enemy.new(name: string, hp: number)
    return setmetatable({ name = name, hp = hp }, Enemy)
end

function Enemy:takeDamage(dmg: number)
    self.hp -= dmg
    if self.hp <= 0 then
        print(`{self.name} died!`)
    end
end

local boss = Enemy.new("Dragon", 500)
boss:takeDamage(120)

Executors

Exploit Tool
⚠️
Using executors in live Roblox games violates Roblox TOS and may result in a permanent ban. This documentation is for educational purposes only.

An executor is a third-party application that injects into the Roblox client process, elevating the script identity level and exposing UNC/sUNC APIs beyond what normal LocalScripts can access.

Identity Levels

LevelContextCapabilities
0Plugin (Studio)Full Studio access
2LocalScriptStandard client APIs only
3CoreScriptRoblox internal UI scripts
6CoreScript+Extended Roblox system APIs
8ExecutorFull UNC/sUNC API access

How Injection Works

1. DLL Injection

Injects a dynamic library into RobloxPlayerBeta.exe to gain execution context.

2. VM Hook

Hooks/replaces the Luau scheduler to elevate script identity to Level 8.

3. API Exposure

Exposes UNC/sUNC globals via the executor's getgenv() environment.

4. Anti-Cheat Bypass

Uses obfuscation or kernel drivers to evade Hyperion/Byfron anti-tamper.

LocalScript

Client-Side

A LocalScript runs exclusively on the client. It can access Players.LocalPlayer, UserInputService, and workspace.CurrentCamera. Changes made here are NOT replicated to the server.

Placement

StarterPlayerScripts, StarterCharacterScripts, StarterGui, or inside a Tool.

LocalPlayer

Has full access to the local player, their character, and PlayerGui.

No server power

Cannot access ServerScriptService or directly modify server-protected data.

Replication

Must use RemoteEvents to communicate server-authoritative changes.

Basic Example

LocalScript · StarterPlayerScripts
local Players = game:GetService("Players")
local UIS = game:GetService("UserInputService")
local localPlayer = Players.LocalPlayer
local character = localPlayer.Character or localPlayer.CharacterAdded:Wait()
local humanoid = character:WaitForChild("Humanoid")

UIS.InputBegan:Connect(function(input, gp)
    if gp then return end
    if input.KeyCode == Enum.KeyCode.E then
        print("E was pressed!")
    end
end)

FireServer (Client → Server)

LocalScript · FireServer
local RS = game:GetService("ReplicatedStorage")
local remote = RS:WaitForChild("BuyItem")

remote:FireServer("Sword", 100)

-- Listen for server reply
local result = RS:WaitForChild("PurchaseResult")
result.OnClientEvent:Connect(function(ok, msg)
    print("Result:", ok, msg)
end)

ServerScript (Script)

Server-Side

A Script runs on the Roblox server. It has authority over game state, datastores, and security logic. It cannot directly interact with any single client's input or screen.

Placement

ServerScriptService is recommended. Also works in workspace (not secure).

Authority

Server is the authority. Always validate RemoteEvent data server-side.

DataStores

Only server scripts can use DataStoreService to persist player data.

All Players

Can access and iterate all connected players via the Players service.

Basic Example

Script · ServerScriptService
local Players = game:GetService("Players")
local DSS = game:GetService("DataStoreService")
local coinStore = DSS:GetDataStore("Coins_v1")

Players.PlayerAdded:Connect(function(player)
    local ok, data = pcall(function()
        return coinStore:GetAsync("uid" .. player.UserId)
    end)
    local coins = (ok and data) or 0
    print(player.Name, "joined with", coins, "coins")
end)

UNC — Unified Naming Convention

UNC

The Unified Naming Convention (UNC) is a community standard defining a common API surface for Roblox script executors. Executors that implement UNC expose the same globally-named functions, making exploit scripts portable across different executors.

ℹ️
UNC tests both the presence AND correct behaviour of each function. A function that exists but returns wrong results is marked fail, not pass.

Result Indicators

IconMeaning
✅ PassExists AND behaves correctly
⛔ FailMissing, or exists but behavior is broken
⏺️ Exists-onlyExists but behavior untested, or optional feature missing
⚠️ Alias missingMain function passes but expected aliases are absent

sUNC — senS' Unified Naming Convention

sUNC

sUNC is a community-maintained fork and evolution of UNC by senS. While it builds on the same executor API concept, sUNC has diverged substantially — removing some UNC functions it considers unreliable, tightening test criteria, and adding entirely new categories like filtergc, replicatesignal, Drawing property accessors, and explicit Encoding functions.

ℹ️
sUNC only tests functions it actively maintains. Functions not listed in sUNC docs are outside its standard — even if they appear in classic UNC.
💡
Run the sUNC test script: loadstring(game:HttpGet("https://script.sunc.su/"))(). As of v2.0 this only runs inside the official sUNC testing game on Roblox.

Key Differences from UNC

AreaUNCsUNC
Base64crypt.base64encode / crypt.base64decodeStandalone base64encode / base64decode functions
LZ4lz4compress / lz4decompress in envExplicit Encoding category
GC filteringgetgc() only (no filter)filtergc() with rich type/value options
Signal replicationNot testedreplicatesignal() tested
Drawing propsDirect property access onlygetrenderproperty / setrenderproperty
Script closuresOptionalgetscriptclosure tested as core
loadstringUnder EnvironmentUnder both Closures and Scripts
rconsoleTestedNot part of sUNC standard

sUNC exposes Base64 and LZ4 as a first-class Encoding category with standalone global functions, distinct from UNC's crypt.* namespace.

FunctionSignatureDescriptionReturns
base64encodebase64encode(str)Encodes a binary or UTF-8 string to Base64. Direct global — no crypt prefix required.string
base64decodebase64decode(b64)Decodes a Base64 string back to its original bytes.string
lz4compresslz4compress(data)Compresses a string with LZ4 block compression. Returns compressed bytes.string
lz4decompresslz4decompress(data, origSize)Decompresses LZ4 data given the original uncompressed size.string
sUNC encoding example
local enc = base64encode("hello world")
print(enc)                          -- "aGVsbG8gd29ybGQ="
print(base64decode(enc))            -- "hello world"

local data = "aaaaaaaaaaaaaaaaaaa"
local compressed = lz4compress(data)
print(lz4decompress(compressed, #data)) -- original string

sUNC — Signals

sUNC

sUNC adds replicatesignal, a function not present in classic UNC, which allows replicating certain client signals to the server.

FunctionSignatureDescriptionReturns
firesignalfiresignal(signal, ...)Fires all connections of an RBXScriptSignal locally with the given arguments.void
getconnectionsgetconnections(signal)Returns a table of connection objects for the signal, each with: Enabled, Function, Thread, Fire, Disconnect, Disable, Enable.table
replicatesignalreplicatesignal(signal, ...)Replicates a signal to the server with the provided arguments. Arguments must match the signal's required structure or an error is thrown. sUNC-exclusive.void
⚠️
replicatesignal requires correct argument types matching the signal. Incorrect arguments throw an error. Consult the Roblox API reference for each signal's expected argument structure.
replicatesignal example
-- Replicate a ClickDetector click to the server
local det = workspace.SomePart.ClickDetector
replicatesignal(det.MouseActionReplicated, game.Players.LocalPlayer, 0)

-- Replicate a GUI MouseWheelForward event
local frame = game.Players.LocalPlayer.PlayerGui.ScreenGui.Frame
replicatesignal(frame.MouseWheelForward, 121, 214) -- x, y required

local ws = WebSocket.connect("wss://ws.postman-echo.com/raw")

ws.OnMessage:Connect(function(message)
    print("Echo:", message)

Server Script Patterns

Script — DataStore save on PlayerRemoving
local Players = game:GetService("Players")
local DSS     = game:GetService("DataStoreService")
local store   = DSS:GetDataStore("PlayerData_v1")

local playerData = {}

Players.PlayerAdded:Connect(function(p)
    local ok, saved = pcall(store.GetAsync, store, p.UserId)
    playerData[p.UserId] = (ok and saved) or { coins = 0, level = 1 }
end)

Players.PlayerRemoving:Connect(function(p)
    local data = playerData[p.UserId]
    if data then
        pcall(store.SetAsync, store, p.UserId, data)
        playerData[p.UserId] = nil
    end
end)

-- Prevent data loss on server shutdown
game:BindToClose(function()
    for uid, data in playerData do
        pcall(store.SetAsync, store, uid, data)
    end
end)
Script — RemoteEvent with server-side validation
local remote = game.ReplicatedStorage:WaitForChild("BuyItem")

remote.OnServerEvent:Connect(function(player, itemName, price)
    -- ALWAYS validate types and sanity on the server
    if type(itemName) ~= "string" then return end
    if type(price) ~= "number" or price < 0 then return end

    local coins = playerData[player.UserId] and playerData[player.UserId].coins or 0
    if coins >= price then
        playerData[player.UserId].coins -= price
        -- grant item to player ...
        game.ReplicatedStorage.PurchaseResult:FireClient(player, true, "Purchased!")
    else
        game.ReplicatedStorage.PurchaseResult:FireClient(player, false, "Not enough coins")
    end
end)
Script — Cross-server messaging
local MSS = game:GetService("MessagingService")

-- Subscribe to a global broadcast topic
MSS:SubscribeAsync("GlobalAnnounce", function(msg)
    for _, player in game.Players:GetPlayers() do
        -- push msg.Data to each player's chat
    end
end)

-- Publish from any server
MSS:PublishAsync("GlobalAnnounce", "Server update in 5 minutes!")
